关于 Tomotoes

我觉得是时候思考一下 Tomotoes 存在的目的了。

最开始的时候，我建立它是想成为一名 高产量，高质量的技术博主。

然而 结果却很可笑，我的技术水平和时间精力远远达不到这一目标。

建站两年，博文少的可怜，原因也很简单在于我自己。

是的，我对待博文的心态出现了问题。

我认为 Tomotoes中的每一篇博文都是我经历的标志，我应该且必须认真对待。

如此往复，每当我想写博客时，总有这个羁绊在我心中–自己能写好这篇文章吗？。

慢慢的，我就不想写了。

So，痛定思痛，我决定了，以后的博文都是随笔体，自己怎么舒服怎么来，不会再咬文嚼字了。

希望我这个强迫症能做到…

关于学业

我决定考研了，想体会一下真正的大学生活。

不过至今为止，目标院校没有决定好，因为现在自己的状态真的是差劲，每天都在玩…

哎，一到家就放飞自我了。

目前自己的 TODO 列表还有一堆待定事项，额，考研先放一放。

反正数学已经一轮完了，啊啊啊，我能行的！

加油，奥力给！

关于爱情

我和 Leann 分手了。

嘿嘿，祝她余生平安喜乐！

目前，我还是无法面对这段经历，先跳过了。

关于 Coding

在 2019 年，我突破了认知的瓶颈。

哈哈，这句话可能有点托大，但确实如此：我不再对任何 PL，框架存在偏见了。

介于以往的经验与认知去了解一项新事物，往往会很容易被蒙蔽住双眼，发现不了其真正的亮点。

是的，我悟出来了，道友，坐。

关于设备

啊哈哈， 前不久入手了一系列设备。 iPad + Pencil + Logitech K380是真滴香！

目前，这篇文章就是在 iPad 的上搞定的～

关于 如何在 iPad 以及 Android 上写博文，我会专门出一篇文章来讲。

这里，先立下个坑。

再聊一聊苹果的应用生态吧。

我之前曾经听说过一句话：

Apple 是史上最伟大的公司，它对抗了整个行业。

哎，自从入手了iPad后，我深刻体会到了这句话的本质。

如果你想开发一个原生App：

  • 你将不得不使用Xcode + Mac
  • 你将不得不使用Objective-C或者Swift
  • 你还需要开发者证书
  • 并且要遵守一系列错综复杂的利它条款
  • 最后还得奢求应用可以过审。

当了解到这一切的时候，我很惊讶。

想找一个可以运行代码的App？对不起，没有。

想找一个可以运行命令的Shell？对不起，没有。

如果有，也只能在Testflight中找到一些应用，来应对。

比如：ish ，不过我的使用经历真的是一言难进。

（感觉，比起Android上的Termux差了好几个层次）

最终，我买了Prompt2。

不装了，ssh 就是了。
