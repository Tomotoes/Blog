项目

  1. hexo-theme-tomotoes (185 star)

    开源的个人网站主题。

    Less JavaScript Ejs Gulp jQuery PWA

    tomotoes 是一款清新精致的 hexo 主题（hexo 是一款文档生成工具）

    本人大二时，精心设计而来，已深度结合了文章打赏、多种评论、百度分析、Google 分析、动态签名、站内搜索等多种插件。

  2. SmartHome (国家组课题/第七届中国软件杯大学生软件设计大赛二等奖)

    智能电器管理综合系统 - 第七届中国软件杯。

    Vue JavaScript Stylus Pug Arduino Tensorflow MongoDB Nginx

    赛题基本介绍:

    1、物联网设备需要将自身数据上传至云端，并支持从云端发起的对物联网设备的控制。

    2、支持收集物联网设备的使用数据进行归纳分析，进一步调整设备使用，并预测未来使用情况。

    当时我正处在大前端的学习当中，从而选择了 Vue + Koa2 来构建一个前后端分离的项目。

    前端使用 Vue 全家桶 + ElementUI + Echarts，后端使用 Koa2 + MongoDB ，智能硬件使用 Arduino + esp8266 开发板。

    物联网通信使用 MQTT 协议，并用 Promise 在应用层撸了一个异步通信框架，前后端通信遵循 REST 规范。服务器使用 Nginx 部署到 Centos7 上，数据加密使用 Bcrypt 和二次加盐的方式。

    其中要准确根据天气信息，条件控制电器是个难题，因此设置每 15 分钟轮询一下天气信息，每天凌晨自动判断，是否过了指令的控制时间。此外，为了满足赛题中要求同时控制多个智能设备的情况，考虑了采用特定 ID 和共用 ID 的方式来单独、共同控制智能设备。

    目前已经把项目开源到了 Github ，里面有完整的项目技术介绍。

    陆陆续续花费两个月的时间从前端到后端再到硬件、数据安全、硬件通信等等，自己独立开发出了项目，并最终取得了全国第二名的好成绩。

  6. scrcpy-gui (913 star)

    Scrcpy 的可视化软件，目前下载量具不完全统计已超过 1w8。

    Electron Vue Adb

    项目简述：

    Scrcpy 是由流行的 Android 模拟器 Genymotion 背后的团队创建的，可以显示和控制通过 USB（或通过 TCP/IP ）连接的 Android 设备，它不需要任何 root 访问权限，它适用于 GNU/Linux 、 Windows 和 MacOS 。

    但官方只提供了命令行一种终端，这在使用中对普通用户会十分的麻烦。

    于是我根据这一痛点，编写了 Scrcpy-Gui 软件。

    解决方案:

    在考虑了 QT,JavaFx 等GUI框架后，我采取了Electron-Vue加ElementUI的方案。

    虽然打包后的软件体积很大，但用户表单中的组件不用再二次开发了，开发效率得到大大提升。

    并且由于需要 实时监测用户的设备是否已连接，我采用了adbkit库，它封装了adb的大部分的功能，结合着 Electron 的进程通信机制，可以很方便的将数据传到前端。

    最终将用户在界面上的配置，生成至 scrcpy 命令，传到后端新建子进程打开即可。
